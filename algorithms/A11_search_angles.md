# A11 — 別の探索角度（鎖の構造を逆利用）

A3/A5 は「先頭 p（＝n）」からのボトムアップ探索だった。鎖の構造を逆から
見ると、別の有力な手法が出る。

## 1. 末尾項（最大項）からの探索

鎖の項は `f_i = m·2^i − 1`（m = p+1）。最大項 `f_{k-1} = m·2^{k-1} − 1` は
他の項よりずっと大きい。逆に「Riesel 数 `R = m·2^{k-1} − 1` が素数なら、
m = (R+1)/2^{k-1} として下位項を復元し、それらも素数か確かめる」ことも
できる。変数の次元は変わらないが、実装によっては大きい項の MR を先に
行って早落ちさせるのに都合が良い（大きい項が合成なら即残り不要）。

## 2. Sophie Germain 素数を「種」に伸ばす

長い第一種鎖は、短い鎖の延長でもある。k=2（Sophie Germain: p, 2p+1）の
既知リストから始めて、各 p について `2p+1, 4p+3, …` を次々 BPSW で試し、
切れるまで伸ばす。既存の SG 素数表（または高速な SG 生成）が「種」になる。
メリット: 下位項が既に素数なので、確認すべきは上位項だけ。既知の長鎖は
ほとんどこの延長で見つかっている。

## 3. p−1 平滑性の利用（Pocklington 的アプローチ）

鎖の項 `f_i = m·2^i − 1` は `m·2^i − 1` の形。m を平滑（多くの小因子）に
選ぶと、各 `f_i` の `f_i + 1 = m·2^i` が平滑になり、**Pocklington / 
Brillhart–Lehmer–Selfridge** のような `N−1` 平滑性を使った素数証明が
安くなる（BPSW より確定的で、証明書付き）。巨大数では「証明」自体が
ボトルネックになりうるので、この形は魅力的。A8 の BPSW を「証明」に
格上げしたい場合の出口。

## 4. 双方向拡張

見つけた長鎖の両端を試す：`f_{-1} = (p−1)/2` が素数なら長さ+1（ただし
p が奇数なので (p−1)/2 が整数・素数なら上に伸ばせる）、`2 f_{k-1}+1` が
素数なら下に伸ばせる。記録更新時の「もう 1 段」を狙う定番テク。

## 5. 確率的早落ち（witness キャッシュ）

A3/A5 の MR は k 項すべてを試す。実は「大きい i の項が合成」なら即終了で
よい。項を i の大きい順（または小さい順）に試し、最初に合成を見つけたら
打ち切る。経験上ほとんどの候補は下位〜中位で落ちるので、平均 MR コストは
k 項分ではなく数項分になる。A3/A5 の `test()` はすでにこの形（i=0 から
順に、偽なら break）。
