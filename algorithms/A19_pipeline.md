# A19 — 篩と MR を分離する（頑健なパイプライン）

巨大数探索は時間がかかるので、途中で落ちても「篩済み候補」を失わない構成にする。

## 2 パス構成

1. **Pass 1（篩）**: A3/A5 で候補 m（または n）を生成し、ファイルへ書き出す。
   各候補は `Int128`/可変長で十分（m 自体はそこまで巨大でないことも多い）。
2. **Pass 2（MR）**: 書き出した候補を BPSW（A8）で本判定。

## メリット

* 篩と MR を別マシン/別実装で走らせられる（MR は GPU、篩は CPU など）。
* 篩結果を保存すれば、MR 実装を差し替えても再篩不要。
* クラッシュ時、篩やり直しではなく MR の続きから再開。

## 候補の圧縮

* 候補は等差数列 `m = r + t·M` なので、`(r, t_range)` だけ保存すればよい。
  全ての m を列挙して書くより遥かに小さい。
* あるいは「生存ビット列」をブロックごとにバイト列で保存（A14 のワード圧縮と相性良）。

## 検証の分離

* Pass 2 の BPSW は「強 MR(2) + 強 Lucas」。偽陽性が事実上ないので、
  見つかった鎖はそのまま記録候補。更に `src/cc_cpu.jl` 等の別実装で
  クロスチェックすれば完全。
